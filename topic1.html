How HTTP Works

HTTP is a client-server protocol where a browser (client) sends a request to a web server, and the server responds with the requested resource such as HTML, images, or data.

Each request includes methods like GET or POST, headers with metadata, and sometimes a body. The server responds with a status code (like 200 OK or 404 Not Found).

Important Features

  • Stateless communication
  • Request/response structure
  • Supports multiple resources per page
